Lesson 1.1: The Basics of the Web and HTML

How the Web Works

The web is a bunch of computers that communicate with each other. A person types in a domain name to access a web page using a computer browser such as Chrome, Opera, Safari, or, when desperate, Internet Explorer. When the person goes to the web page, the computer sends an HTTP Request to a Server. The server finds the appropriate HTML document and sends it back to the user's computer where said web browser interprets the page and displays it on the user's screen.

This Components of the Web Video does a good job of explaining the concept.

HTML

HTML (Hypertext Markup Language) is the standard markup language used to create web pages. HTML documents form the majority of the content on the web. HTML documents contain text content which describes "what you see" and markup which describes "how it looks".

This Intro to HTML Video gives a good overview.

Tags and Elements

HTML documents are made of HTML elements. An element begins with an opening tag and ends with a closing tag. When writing HTML, we tell browsers the type of each element by using HTML tags.

This Into to HTML Tags Video explains the distinction well.

Why Computers are Stupid

Computers are stupid because they interpret instructions literally. This makes them very unforgiving since a small mistake by a programmer can cause huge problems in a program.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them while inline elements appear, well, inline. Go figure. For future reference, span is inline, and div is block.

Lesson 1.3: Adding CSS Style and HTML Structure

The Relationship of CSS to the Web Page

By giving similar HTML elements the same 'class', we can write all the styling for that class just ONCE and it will apply to every element.

Style: The 'style' of an HTML element is just a bunch of rules which describe how that element should look. This style can be specified in a CSS file.

Benefits of Avoiding Repetition

If you give the same style to similar elements, you can change all of them at once just by changing a single line of CSS. If you give the same style to similar elements, you become a #winner when you are able to quickly and easily modify all of them at once just by changing a single line of CSS.

Extra Bonus: This also limits the potential for errors. The final benefit is that it saves a TON of time.

The Relationship Between Indented HTML and Boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box"" is.

Tips: Box Sizing and Equally Fun Four-Sided Stuff

  • You can set the size of a box in px or %. If you used pixels, it is static, so it is not responsive. This can result in a bad user experience.
  • Using flexbox (flexible box) is an efficient way to layout space in a container. It gives you a way to have your HTML elements easily adjust their size based on the size of the page. A complete guide to flexbox can be found here.
  • Divs are block elements as opposed to inline elements, so, by default, they take up the entire width of their parent element. Adding the rule "display: flex;" to the appropriate CSS will override this behavior and let divs appear next to each other.

Lesson 2.1: Introduction to Serious Programming

Important Definitions

  • Computer: A computer is a machine that can carry out instructions given to it by a programmer.
  • Programming: Tells the computer what to do through the use of a precise sequence of steps.
  • Computer Programming Language: A programming language is what programmers use to tell a computer what to do. Python is one example of a programming language.
  • Python: Python is a programming language. When you write Python code and press "Run", a Python Interpreter converts the code you wrote as a set of instructions that the computer itself can understand and execute.
  • Grammar: Grammar refers to the rules that govern what statements are 'legal' in a language. Python has its own grammar.
  • Python Expressions: A Python "expression" is a legal Python statement. For example: print 1 + 1 is a valid expression, but print 1 + (without a number at the end) is not.

Why invent new languages?

Why do we need to invent new languages like Python rather than using natural languages like English or Mandarin?

  1. Ambiguity - We often have different meanings for the same word, and the computer needs specificity.
  2. Verbosity - Computers need a more efficient language.
